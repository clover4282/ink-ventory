# Ink-ventory 작업 규칙

## 기능 변경 순서

사용자 동작, 수집·알림 규칙, 데이터 보관 또는 운영 기능을 추가·변경·삭제할 때 아래 순서를 지킨다.

1. `docs/기능_정의서.md`의 요구사항, 구현 상태와 출시 확인 기준을 먼저 수정한다.
2. 변경된 기능 정의를 기준으로 테스트를 추가하거나 수정한다.
3. 코드와 DB를 최소 범위로 수정한다.
4. 테스트를 실행하고 기능 정의서와 실제 동작이 일치하는지 확인한다.

기능 정의서를 갱신하지 않은 상태에서 사용자 동작을 먼저 변경하지 않는다. 사용자 동작이나 업무 규칙이 바뀌지 않는 순수 리팩터링과 오탈자 수정은 예외로 한다.
